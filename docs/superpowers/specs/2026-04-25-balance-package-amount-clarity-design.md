# 余额包金额展示澄清设计（购买卡片与强制切换弹窗）

## 背景

当前余额包购买区域已经具备基础购买能力，但在金额展示上存在两个明显问题：

1. 余额包卡片底部的人民币金额会被右侧按钮挤压，出现显示不全的情况。
2. 强制切换弹窗里同时出现“当前剩余额度”和“购买金额”，但两者的语义层级不够清晰，用户容易混淆“这是账户里已有的钱”还是“这是本次要付的钱 / 到账的钱”。

这两个问题本质上都属于“金额语义不清 + 横向空间不足”，需要一起处理。

## 目标

1. 保证所有金额完整显示，不再出现被截断、省略或挤压的问题。
2. 明确区分三类金额语义：`当前余额`、`到账金额`、`实付金额`。
3. 让用户在卡片列表和强制切换弹窗里都能一眼分清：
   - 账户当前还剩多少
   - 购买后会到账多少
   - 本次实际需要支付多少
4. 保持现有支付逻辑、下单逻辑、强制切换逻辑不变，仅优化前端信息层级、文案与布局。

## 不在本次范围内

- 不修改后端余额包数据结构。
- 不修改支付创建订单、回调、到账或余额清空逻辑。
- 不调整余额模式业务规则。
- 不重做整页支付流程，仅优化余额包卡片与强制切换弹窗。

## 已确认的用户偏好

### 1. 文案命名

统一使用以下文案：

- `当前余额`
- `到账金额`
- `实付金额`

不再混用“余额 / 额度 / 支付金额 / 购买额度”等相近表达。

### 2. 卡片底部布局

余额包卡片底部采用上下结构：

- 上：`实付金额`
- 下：主按钮

不再保留“左侧金额、右侧按钮”的并排结构。

### 3. 整体设计方向

采用“方案 1：信息分层重排，金额语义完全拆开”。

核心原则：

- 卡片顶部继续强调商品价值（到账金额）
- 卡片底部独立展示购买成本（实付金额）
- 弹窗左侧只讲当前状态，右侧只讲本次购买

## 用户问题总结

### 1. 列表卡片问题

- 卡片底部横向空间不足，`¥xx.xx` 容易被按钮压缩。
- 用户会把卡片顶部 `$xx.xx` 和底部 `¥xx.xx` 当成同一类金额，缺少解释。
- “立即购买”按钮离价格过近，用户更容易先看到动作，忽略金额含义。

### 2. 强制切换弹窗问题

- 左右两块都在展示数字，但没有形成“旧余额 vs 新购买”的清晰对照。
- 用户难以迅速判断：
  - 左边金额是不是还能保留
  - 右边金额是到账金额还是支付金额
  - 购买后实际会发生什么

## 选定方向

采用“**金额语义拆层 + 纵向结算区**”作为本次改版方向。

落点如下：

1. 所有金额都必须带有明确标签，不再让用户依赖上下文猜测。
2. 列表卡片的购买区改成纵向结构，让金额有独立展示空间。
3. 强制切换弹窗按“当前余额 / 购买内容”两大块拆开。
4. 购买内容内部再进一步拆成“到账金额 / 实付金额”，彻底避免混淆。

## 信息层级

### 余额包卡片

优先级从高到低：

1. 线路标签（Codex / General）
2. 余额包名称
3. `到账金额`
4. 支持范围提示
5. `实付金额`
6. 主按钮

说明：

- `到账金额` 代表用户买到的余额价值，应继续作为商品核心信息。
- `实付金额` 属于结算信息，应放在卡片底部独立结算区中。
- 按钮必须依附在结算区内，但不能与金额争抢同一行空间。

### 强制切换弹窗

优先级从高到低：

1. 风险提示（正在切换余额模式）
2. 当前账户状态：`当前余额`
3. 本次购买内容：
   - 目标模式
   - `到账金额`
   - `实付金额`
4. 不可恢复提示
5. 确认按钮

说明：

- 左侧卡片负责解释“你现在有什么”。
- 右侧卡片负责解释“你接下来要买什么”。
- 两块信息不能再共享同一组“金额理解上下文”。

## 页面结构设计

### 1. 余额包卡片

每张卡片保持现有整体框架，但重排底部布局。

#### 顶部区

- 左侧：线路标签、展示标签、余额包名称
- 右侧：`到账金额`

示例：

- `到账金额`
- `$100.00`

#### 中部提示区

继续保留支持范围信息块，例如：

- 仅支持 Codex 分组
- 不可与其他余额模式混用

#### 底部结算区

从当前“左金额 + 右按钮”的横排，改为单独结算块：

- 标题：`实付金额`
- 金额：`¥100.00`
- 下方整行按钮：`立即购买`

设计要求：

- `实付金额` 使用完整展示，不允许 `truncate`。
- 按钮宽度优先跟随容器，不再反向挤压金额。
- 结算区与上方支持范围块之间保留明确视觉间距。

### 2. 强制切换弹窗

弹窗主体保持现有结构，但两张信息卡重新定义职责。

#### 顶部区域

- 保留警告图标
- 保留 `强制切换模式`
- 保留 `切换余额模式？`
- 保留已有说明：当前账户已有其他余额模式余额，继续购买后旧模式余额会被清空

#### 左侧信息卡：当前余额

只展示当前账户状态：

- 标题：`当前余额`
- 当前模式标签（如 `General`）
- 当前余额金额（如 `$1.00`）
- 简短说明：这是当前账户中尚未消耗完的旧模式余额

该卡不展示任何“购买”相关金额。

#### 右侧信息卡：购买内容

只展示本次购买内容：

- 标题：`购买内容`
- 目标模式标签（如 `Codex`）
- 两个并列子卡：
  - `到账金额`
  - `实付金额`
- 下方补充说明：确认购买后，将切换到新模式，新余额到账后旧模式余额会被清空

该卡不展示“当前余额”。

#### 底部风险区

继续保留高风险提示：

- `此操作不可恢复`

#### 操作区

继续保留：

- 次按钮：取消
- 主按钮：确认切换并购买

## 文案规则

本次改版统一采用以下术语：

- `当前余额`：当前账户中已经存在、尚未消耗完的余额
- `到账金额`：购买成功后，将充值进账户的余额数值
- `实付金额`：本次支付时实际需要付款的金额

约束：

1. 列表卡片中，顶部金额统一标注为 `到账金额`。
2. 列表卡片中，底部结算金额统一标注为 `实付金额`。
3. 强制切换弹窗左卡统一使用 `当前余额`。
4. 强制切换弹窗右卡统一使用 `到账金额` 与 `实付金额`。
5. 不再在这几个区域内混用“充值金额 / 购买额度 / 支付金额”等近义表达。

## 响应式与适配

### 1. 卡片列表

- 桌面端：底部结算区采用纵向结构，金额在上、按钮在下。
- 小屏端：保持同样纵向结构，不需要再退回左右并排。

### 2. 强制切换弹窗

- 宽屏：左右两张信息卡并排展示。
- 窄屏：改为上下堆叠，先显示 `当前余额`，再显示 `购买内容`。

目标是避免在窄屏下出现两块内容互相压缩，导致金额和说明再次拥挤。

## 视觉设计原则

1. `到账金额` 使用更偏价值感的强调色，延续当前正向主色表达。
2. `实付金额` 使用更偏结算感的深色正文强调，突出“本次支付”。
3. `当前余额` 保持中性信息展示，不要与“购买内容”抢主视觉。
4. 所有金额标签都要比数值轻一层，但必须始终可见。
5. 不把金额标签做成提示语气，而应做成正式字段标题。

满足以下条件即可视为完成：

1. 余额包卡片中的人民币金额始终完整显示，不再省略。
2. 卡片底部已改为“实付金额在上、按钮在下”的结构。
3. 强制切换弹窗能明确区分：
   - 当前余额
   - 到账金额
   - 实付金额
4. 中文文案统一，不再出现混乱命名。
5. 现有下单与强制切换逻辑行为保持不变。
