# AigoHub 图标重设计设计稿

- 日期：2026-04-21
- 目标：替换当前首页和站点使用的默认图标/Logo 视觉

## 1. 目标
为 AigoHub 设计一枚新的品牌图标，用于替换当前偏“泛科技渐变 SaaS”风格的旧图标。

新图标目标不是解释功能，而是建立更高端、更克制、更像正式品牌的第一印象。

## 2. 已确认方向
- 不保留当前图标里的 `S` 结构
- 不延续现有交织图形语言
- 图标方向偏 **抽象品牌符号**，不是功能图形
- 气质为 **奢一点 + 克制一点**
- 更像高端服务品牌，而不是普通工具站 Logo

## 3. 选定方案
采用 **方案 A：Monogram Seal**

### 关键词
- 品牌徽记感
- 高端
- 克制
- 稳定
- 耐看
- 不依赖炫光和复杂渐变

### 不做的方向
- 不做赛博发光图标
- 不做复杂几何拼接炫技
- 不做明显字母 `S`
- 不做典型 AI 工具站蓝紫渐变符号
- 不做“盾牌 / 门 / Hub / 箭头”这种过于直白的功能隐喻

## 4. 视觉策略

### 4.1 符号策略
图标应为一个抽象、闭合感较强、轮廓清晰的几何徽记。

它应该满足：
- 小尺寸也能辨认
- favicon / 顶栏 / 品牌区都能成立
- 不需要额外说明也显得像正式品牌资产
- 有“稳”和“贵”的感觉，但不装饰过度

### 4.2 造型语言
- 以简洁块面和切角关系为主
- 轮廓优先，内部细节次之
- 允许有轻微切面感，但不能碎
- 更像“印记 / seal / emblem”，而不是动态图标

### 4.3 色彩策略
- 主体避免使用当前旧图标那种高饱和蓝绿渐变语言
- 优先考虑更沉稳的蓝系、墨色、深靛色体系
- 如果使用渐变，也只能非常克制，用于提升质感而不是制造廉价科技感

### 4.4 应用场景
新图标至少要适配：
- 首页左上角品牌位
- 浏览器 favicon / tab icon
- 可能的登录页品牌位
- 浅色 / 深色背景

## 5. 本轮实现边界
本轮优先目标是：
1. 产出新图标方向
2. 替换当前首页使用的图标资源
3. 验证首页显示效果是否成立

本轮不做：
- 不系统重做整套品牌规范
- 不同时扩展完整 logo wordmark 系统
- 不做多套品牌辅助图形

## 6. 验证标准
替换完成后，新图标应满足：
1. 第一眼比旧图标更像正式品牌，而不是 SaaS 模板图标
2. 与当前首页气质更统一
3. 小尺寸显示依然清楚
4. 在浅色和深色背景下都不会显得廉价
