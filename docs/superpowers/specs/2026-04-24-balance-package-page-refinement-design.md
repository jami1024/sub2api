# 余额包购买页高级简洁版优化设计

## 背景

当前余额包购买页已经具备完整业务能力：

- 展示当前余额模式
- 展示余额包商品卡
- 区分兼容与不兼容余额包
- 支持强制切换模式

但当前视觉上仍偏“堆信息”和“轻营销”，与希望中的“高级简洁”方向还有距离。主要问题集中在：

- 页面背景的浅青辉光和卡片发光边缘偏多，显得轻浮
- 顶部账户信息区与“当前余额模式”提示之间存在视觉竞争
- 商品卡内部层级还不够干净，说明区块偏重
- 不可购买卡片更像报错态，而不是受限制的商品态
- 整体更像功能拼装页，而不是成熟产品购买页

本次优化目标不是重做业务逻辑，而是在现有结构上做一次高质量减法，收敛成更克制、更贵、更稳的产品视觉。

## 目标

1. 将余额包购买页整体风格收敛到“高级简洁”而非“轻营销”。
2. 降低背景、边框、发光对视觉的干扰，让商品信息成为主角。
3. 优化顶部账户区，让其承担状态展示而非与商品卡竞争注意力。
4. 强化商品卡的层级与比例，让到账额度、包名、价格、支持范围的阅读顺序更自然。
5. 将“冲突卡片”从警示态收敛为“受限商品态”，降低页面噪声。
6. 保留所有既有业务规则与强制切换逻辑，不改变真实购买行为。

## 不在本次范围内

- 不调整余额包的业务规则
- 不修改支付、下单、回调、到账逻辑
- 不新增新的模式类型
- 不改动订阅页结构
- 不重做整页导航结构

## 设计方向结论

本次确定采用：

- **更高级简洁**（而不是更花或更控制台化）
- 核心方法是：**减法重排**
- 页面角色分工明确：
  - 顶部负责状态
  - 卡片负责商品决策
  - 按钮负责行动

设计关键词：

- 克制
- 清晰
- 留白
- 稳定
- 成熟产品感

## 现状问题拆解

### 1. 背景过重
当前页面大面积浅青背景和发光边缘，让页面看起来像叠加了一层滤镜，卡片与背景之间缺少真正的主次关系。

### 2. 顶部信息失衡
顶部账户卡左侧信息很轻，而右侧“当前余额模式”提示块较重，导致右上角提示比左侧账户本身更显眼，破坏顶部区块的平衡。

### 3. 商品卡节奏偏散
卡片内头部、说明块、价格区、按钮区之间节奏还不够紧凑，信息像分散摆放，缺少统一的阅读顺序。

### 4. 冲突态过像报错
不兼容余额包当前更像警告卡或错误卡，而不是一个“当前受限”的商品。这会让页面显得脏，也会过分放大限制信息。

## 视觉策略

### 1. 背景
背景保留少量浅色氛围，但必须显著降噪：

- 去掉大面积明显的浅青辉光
- 弱化卡片周围的发光边缘
- 让背景只作为衬底，而不是视觉主体

目标：卡片自己形成主视觉，背景只负责承托。

### 2. 顶部账户卡
顶部卡改为更像单层状态条，而不是“左边账户 + 右边强调卡”的双中心结构。

建议结构：

- 左侧：充值账户 / 用户名 / 当前余额
- 右侧：轻量化余额模式状态

余额模式在顶部只作为状态标签，不再做成厚重的大提示块。

### 3. 模式展示
模式仍需要显示，但视觉层级要更低。

建议：

- 使用更轻的 inline 状态样式
- 可以用细字 + 小号 badge 或轻量状态条
- 避免大面积边框和高亮底色

顶部的目标是“告诉用户当前是什么模式”，而不是“强制用户先看模式”。

## 商品卡结构设计

每张余额包卡统一拆为 4 段：

### 1. 头部
- 左侧：模式 badge（Codex / 通用）
- 右侧：到账额度 `$xx.xx`

到账额度仍是首视觉重点。

### 2. 主体
- 商品名称
- 一句简介

标题要更稳，副文案要更短，不允许说明文字抢戏。

### 3. 说明区
- 支持范围
- 不可混用提示

说明区要更轻，不能继续像“卡片里再套一张小卡片”。

### 4. 底部操作区
- 左：充值金额 `¥xx.xx`
- 右：按钮

价格与动作区要底部齐平，保持卡片高度一致时的整体秩序感。

## 商品卡视觉策略

### 可购买卡
可购买卡应表现为：

- 干净边框
- 较轻阴影
- hover 时轻微抬升
- 主按钮清晰但不过分发光

### 不可购买卡
不可购买卡应保留与可购买卡相同骨架，只改变状态感，而不是换一套设计语言。

处理方式：

- 整体轻微降饱和
- 文案变成中性限制说明
- 保留商品结构和信息完整性
- 强制切换按钮作为次按钮，不可抢主视觉

目标：它是一个当前受限商品，而不是报错卡。

## 布局与比例

### 顶部区块
采用单层横向结构，让顶部只承担状态作用，不与下面商品区争主视觉。

### 商品区
桌面端维持两列布局，不建议改成三列。

原因：

- 当前余额包数量不多
- 两列更有“精品商品”感
- 卡片更有呼吸感和稳定感
- 有利于维持高端简洁气质

### 卡片高度
尽量让卡片高度接近一致，底部价格与按钮区域对齐，减少页面高低错落造成的噪声。

## 阅读路径

优化后用户的阅读顺序应是：

1. 顶部看到账户与当前余额
2. 顺带理解当前余额模式
3. 进入商品区比较到账额度
4. 再看包名与支持范围
5. 最后做购买决策

也就是说：

- 顶部说状态
- 卡片说商品
- 按钮说行动

## 组件改造建议

本次优化应继续基于现有余额包页面实现，不需要新增大量组件。

重点修改：

- `frontend/src/views/user/PaymentView.vue`
- `frontend/src/components/payment/BalancePackageCard.vue`
- `frontend/src/components/payment/BalancePackageSummary.vue`

必要时只做轻量提炼，不做与业务无关的大拆分。

## 文案原则

- 模式提示更短、更轻
- 支持范围说明更直接
- 冲突提示更克制
- “强制切换模式”仍需明确风险，但不要把整张卡变成报错盒

## 质量标准

改完后页面应达到：

- 第一眼看起来更干净、更贵、更像成熟产品
- 不再有明显“浅青发光 UI”感
- 顶部与商品区的视觉职责清楚
- 商品卡层级清晰，到账额度与价格更顺
- 受限卡片更统一，不再像单独的报错模块
- 不影响任何既有业务逻辑

## 推荐实施顺序

1. 先降背景和顶部账户卡噪声
2. 再收商品卡边框、说明区与底部操作区
3. 再收冲突卡样式与按钮层级
4. 最后做微调：留白、对齐、按钮统一、响应式检查
