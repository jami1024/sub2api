# AigoHub 首页 Delight 增强设计稿

- 日期：2026-04-21
- 目标页面：`frontend/src/views/HomeView.vue`
- 关联组件：`frontend/src/components/home/AigoHubHeroPanel.vue`

## 1. 目标
在不改变首页核心转化路径的前提下，为默认首页增加“轻松有趣，但仍然可信”的惊喜感。

本轮重点不是重做整页结构，而是把惊喜集中放在首屏 CTA 附近，并增加一个低打扰的隐藏彩蛋，让潜在购买用户在第一眼感到稳、第二眼感到用心。

## 2. 已确认上下文

### 2.1 目标受众
首页主要面向两类人：
- 潜在购买用户：第一次进入页面，快速判断是否值得加微信咨询
- 已有用户：进入首页后，可顺手进入控制台

### 2.2 风格方向
- 整体气质：轻松有趣，但不幼稚
- 优先级：可信优先，其次再显露趣味
- 惊喜主要落点：首屏 CTA 周围 + 隐藏彩蛋

### 2.3 设计原则
- 第一眼应该仍然像一个认真、稳定的服务入口
- 惊喜感来自响应细节，不来自夸张特效
- 所有动画都要短、轻、可忽略，不阻断操作

## 3. 这轮要解决的问题
当前首页虽然已经具备品牌感和转化路径，但仍有这些可提升点：
1. CTA 是“可用的按钮”，但还不是“有回应感的入口”
2. 首屏生命感偏弱，缺少轻度交互反馈
3. 用户点击复制微信后反馈偏功能化，记忆点不足
4. 页面缺少只给探索型用户的小惊喜

## 4. 范围

### 4.1 本轮范围
本轮只做以下增强：
- 首页主 CTA 的悬停联动
- 复制微信成功后的短暂庆祝反馈
- CTA 周边的轻量提示词/漂浮信息
- Hero 面板的轻呼吸感与高光生命感
- 一个不打扰主流程的隐藏彩蛋

### 4.2 本轮不做
- 不改首页主信息架构
- 不改路由、认证、跳转逻辑
- 不引入后端接口改动
- 不增加音效
- 不增加复杂粒子、满屏 confetti、重型鼠标跟随效果
- 不新增依赖

## 5. 方案选择
采用“CTA 做成会回应你的主角”方案。

原因：
- 惊喜和转化动作绑定，价值最高
- 更符合“可信优先”的要求
- 对现有页面结构影响最小，风险可控

## 6. 具体设计

### 6.1 CTA 默认态
主按钮默认仍保持当前清晰、可信、可点击的主动作风格。

默认态要求：
- 外观依然简洁，不做夸张装饰
- 保持“立即加微信”是首屏最明确动作
- 不因为增加 delight 而削弱可读性或专业感

### 6.2 CTA 悬停联动
当鼠标悬停或接近 CTA 时，页面应出现轻量联动反馈：
- CTA 轻微上浮或增强阴影
- CTA 周边 2~3 个提示词出现轻微漂移/位移
- 可叠加一层非常轻的柔和光晕，但不能形成廉价发光按钮感

目标：
- 让按钮看起来“会回应你”
- 让用户感到页面是被认真设计过的
- 不制造打扰，不抢正文信息

### 6.3 CTA 周边提示词
在主 CTA 附近增加 2~3 个轻量提示词，用于强化价值感并提供联动对象。

建议提示词方向：
- 稳定使用
- 省心接入
- 快速咨询

表现要求：
- 默认状态下存在但不抢戏
- 与 CTA 保持明确主次关系
- 可做轻微漂浮、错位、进入或响应动画
- 浅色/深色模式都要成立

### 6.4 复制成功状态
点击 CTA 复制微信号成功后，不仅保留已有复制反馈，还应增加一个短促、克制的成功态。

成功态建议包括：
- 按钮文案短暂切换为成功态，例如“已复制，去微信添加”这类更明确的后续动作提示
- 按钮局部出现一次短暂庆祝反馈，例如柔和扩散、亮点掠过、成功徽记切换等
- 成功态在短时间后自动恢复默认态

约束：
- 总时长应尽量控制在 1 秒级别感知内
- 不阻断连续点击
- 不依赖大型 toast 才能理解结果

### 6.5 Hero 面板生命感
`AigoHubHeroPanel` 不改成大动画展示区，只增加轻微生命感，用于衬托 CTA。

建议增强：
- 极轻的呼吸式浮动
- 局部高光缓慢移动
- 某些状态块保持低频节奏感

目标：
- 让首屏“不死板”
- 但不抢走 CTA 的视觉中心

### 6.6 隐藏彩蛋
在 logo 或 domain badge 上增加一个连续触发型彩蛋，作为探索奖励。

触发思路：
- 用户连续点击若干次后触发
- 出现一句低调、有人味的小提示

彩蛋语气要求：
- 轻松，不油腻
- 不用网络烂梗
- 像认真做产品的人顺手留的小心思

约束：
- 不应误触频繁出现
- 不应遮挡主要操作
- 没发现彩蛋的用户也不会损失任何主流程体验

## 7. 实现边界
优先允许修改：
- `frontend/src/views/HomeView.vue`
- `frontend/src/components/home/AigoHubHeroPanel.vue`
- 如有必要，可补充极少量首页相关测试

实现方式要求：
- 仅使用现有 Vue 3 + TypeScript + Tailwind 风格能力
- 不新增第三方动画库
- 尽量用局部状态和 CSS 动画完成
- 保持代码改动集中、可回滚

## 8. 可用性与动效约束
为了保证“可信优先”，本轮 delight 必须遵守：
- 动效短，不拖操作
- 不让 CTA 乱跳、追鼠标或偏移到难点位置
- 不让页面持续高频闪烁
- 应考虑 `prefers-reduced-motion`，在低动效偏好下退化为更静态的版本

## 9. 验证标准
完成后应满足：

1. 主流程仍然清晰
- 未登录用户仍能直观理解并点击 CTA
- 已登录用户仍能清楚看到进入控制台入口

2. Delight 可感知但不抢戏
- 首屏相比现在更有生命感
- CTA hover / click 成功态具有更强记忆点
- 第一眼印象仍是稳定、可信

3. 彩蛋不打扰
- 正常浏览不受影响
- 探索触发时能感到惊喜，但不会破坏布局和节奏

4. 验证动作
- 运行首页相关测试
- 本地手动验证浅色/深色模式
- 手动验证 CTA hover、点击复制成功态、隐藏彩蛋触发和恢复

## 10. 最小实现版本
第一版只做以下最小可用 delight：
- CTA 悬停联动
- CTA 成功态反馈
- CTA 周边 2~3 个轻量提示词
- Hero 面板轻呼吸
- 一个低调隐藏彩蛋

先不上更重的动效系统，确保这一版先做到“小而准”。
